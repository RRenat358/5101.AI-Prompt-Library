# AGENTS.md — Инструкция для AI-ассистентов и контрибьюторов

Этот файл содержит правила работы с репозиторием `5101.AI-Prompt-Library`.
Следуй им неукоснительно при навигации, поиске, создании и изменении любых файлов.


============================================================
## 1. Структура проекта

- `prompts/` — готовые промпты, сгруппированные по тематическим папкам:
    - `roles/` — ролевые инструкции
    - `psychology/` — психотерапевтические сценарии
    - `text-work/` — обработка и анализ текста
    - //`research/` — поиск, фактчекинг
    - //`coding/` — разработка, код, ревью
- `snippets/` — переиспользуемые фрагменты, «заготовки» (не самостоятельные промпты)
- `guidelines/` — методология, правила и справочники:
    - `style-guide.md` — правила написания и оформления промптов
    - `tagging-system.md` — система тегов (категории, соглашения, примеры)
    - //`evaluation-checklist.md` — чек-лист для проверки качества промпта
    - //`best-practices.md` — продвинутые техники (COT, few-shot и др.)
- //`templates/` — шаблоны для создания новых промптов
- `archive/` — устаревшие версии, эксперименты
- `docs/` — конспекты статей, дополнительные материалы


============================================================
## 2. Именование файлов и директорий

- Язык имён **файлов**: **русский**.
- Язык имён **директорий**: **английский**, нижний регистр, kebab-case.
- Стиль имён файлов: **русский kebab-case** — пробелы заменяются на дефисы.
- Логический разделитель смысловых частей внутри имени: **точка** (`.`), после неё — заглавная буква.
- Служебный постфикс для инструкций, заметок, руководств для человека: `.guide` (строчные латинские буквы).
- Для английских версий промптов: постфикс `.en`.
- Расширение: `.txt` (основное) или `.md` (если необходимо лёгкое форматирование).

**Примеры корректных имён:**

- `Критик-аналитик.txt`
- `Контекстный-снимок.Психотерапия.txt`
- `Объединение-данных-пользователя.guide.txt`
- `prompt-search-expert.en.txt`

**Сортировка и ссылки:** 
имена без пробелов гарантируют целостность URL и корректную работу в любых файловых системах.


============================================================
## 3. Формат файлов промптов

Каждый промпт (и сниппет) может начинаться с мета-блока, 
который **игнорируется LLM** и предназначен для классификации и поиска.

Мета-блок открывается строкой:
`[meta: ignore to ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━]`
(допускается вариативная длина знаков `━`)

Закрывается строкой из знаков `━` (как правило, той же длины, как и в открывающей строке).

Внутри блока пары `ключ: значение` без кавычек:
- `title: Человекочитаемое название`
- `description: Краткое описание сути`
- `version: YYYYMMDD-HHMM`
- `tags: [#tag1, #tag2]`
- `target_models: [gpt-4o, claude-3.5-sonnet]` (опционально)
- `status: draft | tested | ready | archived`

**AI-ассистент должен:**
- Распознавать мета-блок по открывающей конструкции `[meta: ignore to`.
- Извлекать из него все перечисленные ключи.
- Использовать их для поиска, фильтрации и понимания контекста.
- Не включать содержимое мета-блока в цитируемый текст промпта при передаче в LLM.

Если мета-блок отсутствует, файл считается неклассифицированным.


============================================================
## 4. Система тегов

Все теги и правила их назначения описаны в файле `guidelines/tagging-system.md`.
AI обязан обращаться к этому файлу при любых операциях, связанных с тегами (добавление, удаление, поиск по тегам).


============================================================
## 5. Правила создания и редактирования промптов

При создании нового промпта или сниппета:

1. Определи тематическую категорию и помести файл в соответствующую папку (`prompts/...` или `snippets/`).
2. Именуй файл строго по правилам из раздела 2.
3. Добавь мета-блок в соответствии с разделом 3.
4. Оформи содержимое согласно стилю, описанному в `guidelines/style-guide.md`.
5. Если требуется пояснение для человека, создай одноимённый файл с постфиксом `.guide.txt`.
6. Проверь промпт по чек-листу `guidelines/evaluation-checklist.md`.
7. При необходимости используй продвинутые техники из `guidelines/best-practices.md`.


============================================================
## 6. Для AI: как анализировать и искать по репозиторию

- Первым делом сканируй мета-блоки файлов.
- Используй поля `tags`, `description` и `title` для поиска.
- При отсутствии мета-блока анализируй имя файла и его содержимое.
- Ссылки на внешние правила всегда бери из `guidelines/`.
- Сомневаешься в категории или именовании — спроси у пользователя.
