---
**Назначение**
Этот ПРОМТ предназначен для **объединения уже структурированных данных** клиента из нескольких файлов парсинга психотерапевтических сессий. Этот ПРОМТ работает с **уже извлеченными и категоризированными данными**, сохраняя все нюансы психологического профиля.

**Контекст: Психотерапевтические данные**
ВСЕ обрабатываемые данные относятся к психотерапевтическому контексту.
Это не просто данные - это психологический портрет человека с:
- Травматическим опытом
- Внутренними конфликтами
- Эмоциональными состояниями
- Межличностными динамиками

🟪 **1. УПРАВЛЯЮЩАЯ КОНФИГУРАЦИЯ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**[НАСТРОЙКА РОЛЕЙ]**
При анализе файлов используй следующие обозначения:
- **Пользователь/Клиент:** ____ (Клиент - по умолчанию) - тот, чьи данные извлекаются.
- **Собеседник:** ____ (Психотерапевт - по умолчанию).
Если в заголовках файлов использованы другие термины - сохраняй их для целостности данных.

**[ФОРМАТ ВХОДНЫХ ДАННЫХ И ИДЕНТИФИКАЦИЯ БАЗЫ]**

1. **ОПРЕДЕЛЕНИЕ ТИПА ФАЙЛОВ:**
   - Файл считается **БАЗОЙ ДАННЫХ**, если содержит маркеры `[БАЗА ДАННЫХ КЛИЕНТА]` в начале и `[КОНЕЦ БАЗЫ]` в конце, с соответствующим содержимым между ними.
   - Файл считается **СТРУКТУРИРОВАННЫМИ ДАННЫМИ** (парсингом), если содержит блоки `КОНСТАНТНЫЕ ДАННЫЕ` и `ДИНАМИЧЕСКИЕ ДАННЫЕ` в формате, соответствующем разделу 7 данного ПРОМТа, но без маркеров базы.

2. **СЦЕНАРИИ ОБЪЕДИНЕНИЯ:**

   **СЦЕНАРИЙ А: БАЗА + СТРУКТУРИРОВАННЫЕ ДАННЫЕ**
   - Если один файл имеет маркеры базы, а другой - нет, то:
     * Файл с маркерами считается **ТЕКУЩЕЙ БАЗОЙ**.
     * Файл без маркеров считается **НОВЫМИ ДАННЫМИ** для объединения.

   **СЦЕНАРИЙ Б: ДВА ФАЙЛА БЕЗ МАРКЕРОВ (оба структурированные данные)**
   - Если оба файла не содержат маркеров базы:
     * **Определение базы:** Проанализируй метаданные:
       1. Имена файлов: файл с названием, содержащим "база", "base", "основа", "итог", "объединение" или аналогичные термины, считается базой.
       2. Даты в именах файлов: файл с более ранней датой (если указано в формате YYYYMMDD) считается базой.
       3. Объем данных: файл с большим количеством информации (больше пунктов, более полная структура) считается базой.
       4. Порядок предоставления: если невозможно определить иначе, первый предоставленный файл считается базой.
     * Второй файл считается **НОВЫМИ ДАННЫМИ**.

   **СЦЕНАРИЙ В: ДВА ФАЙЛА С МАРКЕРАМИ БАЗЫ**
   - Если оба файла содержат маркеры базы:
     * Это считается ошибкой формата. Прекрати обработку и верни сообщение: "ОШИБКА: Оба файла содержат маркеры базы. Укажите явно, какой файл является текущей базой, а какой - новыми данными."

   **СЦЕНАРИЙ Г: НЕКОРРЕКТНЫЙ ФОРМАТ**
   - Если файл не соответствует ни одному из форматов (нет ни маркеров базы, ни структурированных блоков КОНСТАНТЫ/ДИНАМИКА):
     * Прекрати обработку и верни сообщение: "ОШИБКА: Файл [имя файла] имеет некорректный формат. Ожидается либо файл базы с маркерами [БАЗА ДАННЫХ КЛИЕНТА]...[КОНЕЦ БАЗЫ], либо структурированные данные с блоками КОНСТАНТНЫЕ ДАННЫЕ и ДИНАМИЧЕСКИЕ ДАННЫЕ."

3. **ИЗВЛЕЧЕНИЕ ДАННЫХ ИЗ БАЗЫ:**
   - Для файла с маркерами базы: извлеки все содержимое между `[БАЗА ДАННЫХ КЛИЕНТА]` и `[КОНЕЦ БАЗЫ]`.
   - Удали сами маркеры из извлеченного текста.

4. **ОБЪЕДИНЕНИЕ НЕСКОЛЬКИХ ФАЙЛОВ:**
   - При наличии более двух файлов: последовательно объединяй их, используя результат предыдущего объединения как новую базу для следующего файла.
   - Для последовательного объединения: после каждого шага результат автоматически становится файлом базы (с маркерами) для следующего шага.

5. **ЯВНОЕ УКАЗАНИЕ (РЕКОМЕНДУЕМЫЙ ПОДХОД):**
   - Для избежания неоднозначностей, рекомендуется предоставлять файлы с явными указаниями:
     * Базовый файл должен содержать маркеры `[БАЗА ДАННЫХ КЛИЕНТА]` и `[КОНЕЦ БАЗЫ]`.
     * Файл с новыми данными должен быть чистым структурированным файлом без маркеров.

🟦 **2. ПРИНЦИПЫ ОБРАБОТКИ ПСИХОЛОГИЧЕСКИХ ДАННЫХ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**[КЛЮЧЕВЫЕ ПСИХОЛОГИЧЕСКИЕ ПРИНЦИПЫ]**

**Сохраняй все психологические нюансы:**
1. **Противоречия как отражение амбивалентности:**
   - В психологических данных противоречивые утверждения часто отражают внутренние конфликты, амбивалентность или развитие взглядов.
   - НИКОГДА не удаляй противоречивые данные, если они не являются явными ошибками.

2. **Эволюция переживаний:**
   - Чувства, убеждения и состояния могут меняться со временем.
   - Сохраняй хронологическую или логическую последовательность, когда это возможно.

3. **Контекстуальная важность:**
   - Даже похожие формулировки могут иметь разный смысл в разных контекстах.
   - Сохраняй уникальные формулировки, метафоры, идиомы.

**[СПЕЦИФИКА ПСИХОТЕРАПЕВТИЧЕСКИХ ДАННЫХ]**

**Особое внимание уделяй:**
1. **Травматическим переживаниям:** формулировки должны сохраняться максимально близко к оригиналу.
2. **Защитным механизмам:** рационализация, проекция, отрицание могут создавать видимые противоречия.
3. **Терапевтическим соглашениям:** фразы, с которыми клиент согласился, имеют особый статус.
4. **Телесным ощущениям:** "боль в груди", "ком в горле" - это важные диагностические данные.
5. **Метафорам и образам:** "камень на сердце", "проваливаюсь в пустоту" - сохраняй дословно.

🟩 **3. ПРАВИЛА ОБЪЕДИНЕНИЯ СТРУКТУРИРОВАННЫХ ДАННЫХ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**[ОБЩИЙ АЛГОРИТМ]**
1. **Идентифицируй блоки:**
   - Раздели каждый файл на **КОНСТАНТНЫЕ ДАННЫЕ** и **ДИНАМИЧЕСКИЕ ДАННЫЕ**.
   - Сохрани оригинальную структуру кластеров.

2. **Обработай КОНСТАНТНЫЕ ДАННЫЕ:**
   - **Объективные факты:** возраст, даты, состав семьи, материальные условия.
   - **Правила объединения:**
     * **Добавление:** Новые факты добавляй в соответствующие кластеры.
     * **Дубликаты:** При полном совпадении текста оставляй один вариант.
     * **Уточнения:** Если факт повторяется с дополнительными деталями, используй более полную формулировку.
     * **Противоречия:** Сохрани оба варианта, но отметь в комментарии (см. правила ниже).

3. **Обработай ДИНАМИЧЕСКИЕ ДАННЫЕ:**
   - **Субъективные переживания:** чувства, убеждения, страхи, надежды.
   - **Правила объединения:**
     * **Добавление:** Все новые пункты сохраняй.
     * **Эволюция:** Если похожие утверждения показывают развитие мысли, сохрани последовательность.
     * **Амбивалентность:** Противоречивые чувства сохраняй как отражение внутреннего конфликта.

**[ДЕТАЛЬНЫЕ ПРАВИЛА ДЛЯ РАЗНЫХ ТИПОВ ДАННЫХ]**

**Для Биографических данных (Константы):**
- Возраст, даты: при противоречии сохрани оба варианта, но укажи источник.
- Семейный состав: объединяй информацию, дополняя деталями.
- Хронология событий: создай единую временную линию.

**Для Эмоциональных состояний (Динамика):**
- Интенсивность: "сильная депрессия" + "усиливающаяся депрессия" → "депрессия, описываемая как сильная и усиливающаяся".
- Нюансы: разные аспекты одного состояния сохраняй отдельно.

**Для Отношений (и Константы, и Динамика):**
- Факты о отношениях (кто, когда) → в Константы.
- Чувства и восприятия → в Динамику.
- Противоречивые оценки отношений сохраняй как многогранность.

**Для Убеждений и ценностей:**
- Религиозные аспекты: особо бережно сохраняй формулировки.
- Самооценка: противоречивые утверждения ("я плохой" + "я хороший") отражают самооценочный конфликт.

**[ОБРАБОТКА ПРОТИВОРЕЧИЙ]**

**Система приоритетов:**
1. **Явные исправления:** Если в данных есть пометки "исправляю", "точнее", "на самом деле" → заменяй старую информацию.
2. **Хронологический приоритет:** Более поздние данные могут уточнять ранние.
3. **Контекстуальная полнота:** Более подробное описание предпочтительнее краткого.

**Маркировка противоречий (только в процессе работы, не в итоговом файле):**
- Если факты противоречат без возможности разрешения, сохрани оба, но в итоговом файле представь их как разные пункты.
- Пример: "Развелся 1,5 года назад" и "Развелся 2 года назад" → оставь оба, если нет явного исправления.

**Пример обработки противоречия:**
- В базе: «– Развелся 1,5 года назад».
- В новых данных: «– Развелся год назад».
- Результат объединения: «– Развелся 1,5 года назад (в других данных указано: год назад).»

**[СТРУКТУРИРОВАНИЕ]**

**Кластеры:**
1. **Объедини схожие кластеры** из разных файлов.
2. **Создавай новые кластеры** при появлении значительного количества данных по новой теме.
3. **Минимальный размер кластера:** 2-3 пункта. Меньшее количество объединяй в существующие категории.

**Внутри кластеров:**
- **Хронологический порядок** для Констант.
- **Логический порядок** для Динамики: от общих убеждений к конкретным переживаниям.
- **Тематические подгруппы** при наличии 4+ пунктов по подтеме.

**Форматирование:**
- Сохрани оригинальное форматирование (жирный, курсив, эмодзи).
- Приведи маркеры пунктов к единому виду: используй тире `–` (U+2013) для всех пунктов.
- Приведи символы кластеров к единому виду: используй, как в пункте данной инструкции "8. ФОРМАТ ИТОГОВОГО ФАЙЛА"

🟨 **4. ПРАКТИЧЕСКИЙ АЛГОРИТМ ВЫПОЛНЕНИЯ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**ПЕРЕД НАЧАЛОМ:**
- Если предоставлено два файла — обрабатывай их как пару (база + новые данные).
- Если файлов больше двух — обрабатывай их последовательно: объедини первые два, результат станет базой для третьего, затем объедини результат с четвёртым и т.д.
- Если передан только один файл — выведи сообщение об ошибке: «ОШИБКА: Для объединения необходимо минимум два файла.»

**ШАГ 0: ИДЕНТИФИКАЦИЯ И ИЗВЛЕЧЕНИЕ ДАННЫХ**
1. Определи тип каждого файла согласно правилам из раздела [ФОРМАТ ВХОДНЫХ ДАННЫХ И ИДЕНТИФИКАЦИЯ БАЗЫ].
2. Если выявлен некорректный формат - прекрати обработку и верни сообщение об ошибке.
3. Определи, какой файл является БАЗОЙ, а какой - НОВЫМИ ДАННЫМИ.
4. Извлеки данные из БАЗЫ (удали маркеры, если есть).
5. Извлеки данные из НОВЫХ ДАННЫХ:
   - Если файл имеет маркеры `[БАЗА ДАННЫХ КЛИЕНТА]` и `[КОНЕЦ БАЗЫ]`, извлеки содержимое между ними (игнорируя сами маркеры).
   - Если файл не имеет маркеров — используй всё содержимое файла.

**ШАГ 1: АНАЛИЗ ИСХОДНЫХ ФАЙЛОВ**
1. Прочитай оба файла полностью.
2. Выдели основные блоки: Константы и Динамика.
3. Составь список всех кластеров в каждом файле.
4. Отметь уникальные кластеры, отсутствующие в другом файле.

**ШАГ 2: ОБЪЕДИНЕНИЕ КОНСТАНТ**
1. Возьми за основу кластеры из Базового файла.
2. Для каждого кластера из Дополнительного файла:
   - Если есть аналогичный кластер → объедини пункты.
   - Если кластер отсутствует → добавь новый кластер.
3. Внутри кластеров:
   - Проверь каждый пункт на дублирование.
   - При противоречии используй правила из раздела 3.
   - Объедини схожие пункты с сохранением всех деталей.

**ШАГ 3: ОБЪЕДИНЕНИЕ ДИНАМИКИ**
1. Сохрани все уникальные пункты из обоих файлов.
2. Группируй по темам, создавая логическую структуру.
3. Обрати особое внимание на:
   - Эмоциональные состояния: объединяй описания одного состояния.
   - Отношения: сохрани все нюансы восприятия.
   - Убеждения: противоречия как отражение внутренней работы.

**ШАГ 4: ФИНАЛЬНАЯ СТРУКТУРИЗАЦИЯ**
1. Проверь целостность данных.
2. Удали технические дубликаты (полное совпадение текста).
3. Упорядочи кластеры по значимости для психологического портрета.
4. Сохрани все оригинальные формулировки клиента.

**ШАГ 5: ВАЛИДАЦИЯ**
Перед финальным выводом проверь:
1. Сохранены ли все уникальные метафоры и образы?
2. Отражена ли амбивалентность и внутренние конфликты?
3. Сохранена ли хронология событий и переживаний?
4. Не потеряны ли нюансы из-за излишнего обобщения?

🟪 **5. ОСОБЫЕ СЛУЧАИ И ИСКЛЮЧЕНИЯ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**Случай 1: Противоречивые диагностические данные**
Если в разных файлах указаны разные диагнозы или симптомы:
- Сохрани все варианты.
- Добавь пометку о контексте (если известен).

**Случай 2: Изменение терапевтической позиции**
Если клиент меняет отношение к каким-то событиям:
- Сохрани оба отношения как этапы терапевтической работы.
- Не пытайся определить, какое отношение "правильное".

**Случай 3: Цитаты и прямая речь**
- Сохрани все цитаты дословно.
- Особо отметь цитаты значимых других (матери, отца, супруга).

**Случай 4: Культурные и религиозные особенности**
- Религиозные формулировки сохраняй без изменений.
- Культурный контекст учитывай при интерпретации семейных отношений.

🟦 **6. ПРЕДУПРЕЖДЕНИЯ И ОГРАНИЧЕНИЯ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**Чего НЕ делать:**
1. **НЕ** упрощай сложные переживания.
2. **НЕ** удаляй "неудобные" или противоречивые данные.
3. **НЕ** делай клинических выводов или диагнозов.
4. **НЕ** меняй эмоциональную окраску высказываний.
5. **НЕ** объединяй данные, если это искажает смысл.

**Критерии успешного объединения:**
1. Психологическая целостность портрета сохранена.
2. Все нюансы и противоречия отражены.
3. Структура удобна для терапевтической работы.
4. Данные пригодны для дальнейшего анализа.

⚠️ **7. ПРОТОКОЛ ЦЕЛОСТНОСТИ ДАННЫХ ПРИ ВЫВОДЕ**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

**[СИСТЕМА МАРКЕРОВ ДЛЯ КОНТРОЛЯ]**
1. **После каждого тематического кластера** ставь маркер: `[КЛАСТЕР: "название кластера" ЗАВЕРШЕН]`
2. **После завершения каждого основного раздела** ставь маркер: `[РАЗДЕЛ: "название раздела" ЗАВЕРШЕН]`
3. **После полного завершения всех данных** ставь маркер: `[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]`

**[ПРАВИЛА РАССТАНОВКИ МАРКЕРОВ]**
1. **Маркер кластера:** Ставь после последнего пункта кластера, на отдельной строке.
2. **Маркер раздела:** Ставь после завершения всех кластеров раздела, на отдельной строке.
3. **Финальный маркер:** Ставь в самом конце, после всех данных.
4. **Маркировка кластеров с подгруппами:** Маркер `[КЛАСТЕР: "название" ЗАВЕРШЕН]` ставится только для основного кластера, после всех его подгрупп и пунктов. Для подгрупп маркеры не ставятся.

**[ВАЖНОЕ ПРИМЕЧАНИЕ]**
- Название кластера в маркере должно точно соответствовать названию в заголовке (без форматирования).
- Пример: Для кластера `🔸 [**Биографические данные**]` маркер будет: `[КЛАСТЕР: "Биографические данные" ЗАВЕРШЕН]`

**[ПРАВИЛА ОБНАРУЖЕНИЯ ОБРЫВА]**
- Если маркер `[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]` отсутствует → вывод был прерван.
- Если последний кластер в разделе не имеет маркера завершения → раздел неполный.
- Если раздел не имеет своего маркера завершения → данные обрезаны.

✨ **[ИНСТРУКЦИЯ ДЛЯ ПОЛЬЗОВАТЕЛЯ]** ✨
Если вывод прерван (нет маркера `[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]`):
1. Скопируйте **весь** полученный текст (включая все маркеры и структуру).
2. В новом запросе напишите: **«Продолжить с того места, где остановился вывод. Вот что уже выведено:»** и вставьте скопированный текст.
3. Модель продолжит генерацию с того места, где был последний завершённый кластер или раздел.

**Важно:** Модель не сохраняет контекст между запросами автоматически. Для продолжения необходимо явно передавать уже сгенерированные данные.

🟥 **8. ФОРМАТ ИТОГОВОГО ФАЙЛА**
≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡≡

Используй строго следующую структуру и обозначения для представления **ОБЪЕДИНЁННЫХ ДАННЫХ** с соблюдением протокола целостности:

"""
==================================================
🟧 **КОНСТАНТНЫЕ ДАННЫЕ**
==================================================
(Константы. Объективная основа: факты, история, неизменные параметры)

🔸 [**Тематический кластер 1**]
    **Подгруппа (если есть):**
      – ...
      – ...
    – ...
   [КЛАСТЕР: "Тематический кластер 1" ЗАВЕРШЕН]
🔸 [**Тематический кластер 2**]
    – ...
   [КЛАСТЕР: "Тематический кластер 2" ЗАВЕРШЕН]

[РАЗДЕЛ: "КОНСТАНТНЫЕ ДАННЫЕ" ЗАВЕРШЕН]

==================================================
🔄 **ДИНАМИЧЕСКИЕ ДАННЫЕ**
==================================================
(Динамика. Субъективное поле: опыт, цели, боли, переживания, изменения)

🔹 [**Тематический кластер 1**]
    **Подгруппа (если есть):**
      – ...
      – ...
    – ...
   [КЛАСТЕР: "Тематический кластер 1" ЗАВЕРШЕН]
🔹 [**Тематический кластер 2**]
    – ...
   [КЛАСТЕР: "Тематический кластер 2" ЗАВЕРШЕН]

[РАЗДЕЛ: "ДИНАМИЧЕСКИЕ ДАННЫЕ" ЗАВЕРШЕН]

[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]
"""

**Требования к итоговому выводу данных:**
1. Полнота: все данные из исходных файлов должны быть представлены.
2. Структурированность: четкое разделение на Константы и Динамику.
3. Читаемость: логичная группировка, понятные заголовки.
4. Психологическая адекватность: данные отражают сложность внутреннего мира клиента.
5. Целостность: наличие всех маркеров завершения.
6. Ответ с ДАННЫМИ выводи в текстовом блоке.

---
**Цель:** Создать единую, полную, структурированную базу данных клиента, которая сохраняет всю сложность и богатство психологического материала, собранного за несколько терапевтических сессий.

**Принцип:** "Лучше сохранить противоречие, чем потерять нюанс". В психотерапии именно противоречия часто являются наиболее ценным материалом для работы.
---
Примени ПРОМТ к загруженным файлам.
Ответ выведи в текстовом блоке.
