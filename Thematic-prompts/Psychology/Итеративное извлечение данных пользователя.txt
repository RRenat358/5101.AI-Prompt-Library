[НАСТРОЙКА РОЛЕЙ]
При анализе диалога используй следующие термины(обозначения) участников:
- Пользователь: ____ (Клиент - по умолчанию) - тот, чьи данные извлекаются.
- Собеседник: ____ (Психотерапевт - по умолчанию) - тот, кто ведёт диалог.
- Остальные участники: ____ ("как есть" в диалоге - по умолчанию).
Если тот, кто даёт тебе этот ПРОМТ, ввёл для роли(ей) свои термины - используй их, иначе - термины по умолчанию.
Все формулировки в инструкции адаптируй под эти общие термины.
ВАЖНО: В итоговом ответе все утверждения должны быть изложены от третьего лица, в соответствии с выбранным термином для пользователя. Не используй местоимения "я", "мне" от лица пользователя в финальной структурированной выгрузке.

[ФОРМАТ ВХОДНЫХ ДАННЫХ И ИНИЦИАЛИЗАЦИЯ]
1. **БАЗА ДАННЫХ КЛИЕНТА** (опционально, для итеративного режима):
   - Если в начале текста присутствует блок, начинающийся с `[БАЗА ДАННЫХ КЛИЕНТА]` и заканчивающийся `[КОНЕЦ БАЗЫ]`, извлеки оттуда структурированные данные.
   - Этот блок содержит результаты предыдущих парсингов в стандартном формате (Константы и Динамика).
   - Сохрани эти данные как **ТЕКУЩАЯ БАЗА** для последующего объединения.

2. **НОВЫЙ ДИАЛОГ** (обязательно):
   - Текст, расположенный после блока `[КОНЕЦ БАЗЫ]` (или весь текст, если блока базы нет), считается **НОВЫМ ДИАЛОГОМ** для анализа.
   - К этому тексту будут применены [ОСНОВНЫЕ ПРАВИЛА ИЗВЛЕЧЕНИЯ] из раздела 2.

[ПРАВИЛА ОБЪЕДИНЕНИЯ ДАННЫХ]
- Если ТЕКУЩАЯ БАЗА загружена, используй ее как основу для результата.
- Примени [ОСНОВНЫЕ ПРАВИЛА ИЗВЛЕЧЕНИЯ] к НОВОМУ ДИАЛОГУ, получи **НОВЫЕ ДАННЫЕ**.
- **Объединяй** данные следующим образом:
  - **Константы:**
    * Новые факты добавляй.
    * Если факт в НОВЫХ ДАННЫХ противоречит факту в ТЕКУЩЕЙ БАЗЕ, проверь, есть ли в НОВОМ ДИАЛОГЕ явное исправление (по правилам из [ОСОБЫЕ ПРАВИЛА ОБРАБОТКИ]). Если да — заменяй старую запись новой. Если нет — сохрани оба варианта, но в итоговой выгрузке отдавай приоритет НОВЫМ ДАННЫМ (они могут быть уточнением).
  - **Динамика:**
    * Новые переживания, убеждения, чувства добавляй к существующим.
    * Противоречивые утверждения сохраняй как разные состояния/мнения, возникшие в разное время.
    * **ФИЛОСОФИЯ:** "Лучше сохранить противоречие, чем потерять нюанс". В психотерапии именно противоречия часто являются наиболее ценным материалом для работы.

[СИСТЕМА МАРКЕРОВ ДЛЯ КОНТРОЛЯ КАЧЕСТВА]

ПРИНЦИП: Пользователь маркирует проверенные данные, модель работает с немаркированными.

**МАРКЕРЫ:**
• [+] — ПРОВЕРЕННЫЙ ЭТАЛОН. НЕ изменять, НЕ удалять, НЕ объединять.
• [_] — СЫРОЙ ПУНКТ. Можно обрабатывать, объединять, улучшать.

**ФОРМАТ:**
[статус][дата] Текст пункта.
Дата: [YYYYMMDD] (8 цифр) или [________] (8 подчёркиваний).
Даты устанавливаются пользователем. Наличие даты фиксирует привязку времени.

**ПОРЯДОК ВЫВОДА ВНУТРИ КЛАСТЕРА:**
1. Все пункты с датами, отсортированные по дате (возрастание)
   - Внутри каждой даты: сначала [+], затем [_]
2. Все пункты без дат ([________])
   - В том же порядке: [+], затем [_]

**ОБРАБОТКА НОВЫХ ДАННЫХ:**
- Все утверждения, извлечённые из нового диалога, оформляются как [_][________].
- При объединении с существующей базой сохраняются все маркеры [+], установленные пользователем.
- Существующие пункты [_] могут быть оптимизированы согласно правилам ниже.

**ПРАВИЛА ОБЪЕДИНЕНИЯ:**
1. Автоматически объединяются ТОЛЬКО:
   - Точные дубликаты [_][________] → один [_][________]
   - Похожие пункты [_][________] (любое количество) → один [_][________]
   - Пункты [_] с ОДИНАКОВОЙ датой → один [_] с той же датой

2. В блок ПРЕДЛОЖЕНИЙ отправляются:
   - Группы с пунктами [_], имеющими РАЗНЫЕ даты
   - Смешанные группы ([_] с датой + [_] без даты)
   - Любые группы, содержащие хотя бы один [+]

3. Новые данные из диалога → всегда [_][________].
Существующие маркеры в базе сохраняются согласно правилам системы маркеров.

**Логика принятия решений (псевдокод):**
if (все пункты == [_][________])
    объединять в один [_][________]
else if (даты одинаковые)
    объединять, сохраняя дату
else if (даты разные)
    создавать предложение
else if (в группе есть пункты с датой и без даты)
    создаватьпредложение
else if (в группе есть хотя бы один [+])
    создавать предложение

**БЛОК ПРЕДЛОЖЕНИЙ:**
- Создаётся после основного вывода данных
- Содержит готовые варианты разрешения конфликтов
- Используются разделители:
  * "=====" между разными предложениями
  * "↓↓↓" между конфликтными пунктами и предлагаемым вариантом
- Предлагается только один лучший вариант на усмотрение модели

[ПРОТОКОЛ ЦЕЛОСТНОСТИ ДАННЫХ ПРИ ВЫВОДЕ]
**Важная техническая особенность:** Из-за возможных ограничений на длину ответа, вывод данных может быть прерван. Для контроля целостности и возможности продолжить вывод используется система маркеров:
1. **После каждого тематического кластера** ставь маркер: `[КЛАСТЕР: "название кластера" ЗАВЕРШЕН]`
2. **После завершения каждого основного раздела** ставь маркер: `[РАЗДЕЛ: "название раздела" ЗАВЕРШЕН]`
3. **После полного завершения всех данных** ставь маркер: `[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]`

✨ **[ИНСТРУКЦИЯ ДЛЯ ПОЛЬЗОВАТЕЛЯ]** ✨
Если вывод прерван (нет маркера `[ВЫВОД ДАННЫХ ПОЛНОСТЬЮ ЗАВЕРШЕН]`):
1. Скопируйте **весь** полученный текст (включая все маркеры и структуру).
2. В новом запросе напишите: **«Продолжить с того места, где остановился вывод. Вот что уже выведено:»** и вставьте скопированный текст.
3. Модель продолжит генерацию с того места, где был последний завершённый кластер или раздел.

**Важно:** Модель не сохраняет контекст между запросами автоматически. Для продолжения необходимо явно передавать уже сгенерированные данные.

[КЛЮЧЕВОЕ ПРАВИЛО ВЫВОДА]
В финальном ответе должны присутствовать ТОЛЬКО данные, которые:
1. Были прямо высказаны пользователем от первого лица.
2. Являются формулировками собеседника (или другого лица), с которыми пользователь **явно и недвусмысленно выразил согласие**.
3. **Критерии явного согласия:**
   a. **Допустимые маркеры:** Прямые утвердительные ответы в том же или следующем сообщении пользователя: «да», «верно», «точно», «согласен», «это так», «именно», «всё так», «правильно», «я согласен», «вы правы».
   b. **Недопустимые реакции (НЕ согласие):**
      * Встречные вопросы: «Это верно?», «Почему вы так думаете?».
      * Выражения сомнения или вероятности: «наверное», «возможно», «может быть», «не уверен».
      * Развитие темы или рассказ новой истории без прямого подтверждения слов собеседника.
      * Молчание (отсутствие ответа).
   c. **Запрет на домысел:** Если согласие не выражено через допустимые маркеры (п.3a), формулировка собеседника **НЕ включается в базу**. Запрещено интерпретировать иные реакции как скрытое или косвенное согласие.

[ПРИНЦИПЫ ОБРАБОТКИ ПСИХОЛОГИЧЕСКИХ ДАННЫХ]
**Контекст: Психотерапевтические данные**
Все обрабатываемые данные относятся к психотерапевтическому контексту. Это не просто данные - это психологический портрет человека с травматическим опытом, внутренними конфликтами, эмоциональными состояниями и межличностными динамиками.

**Ключевые психологические принципы:**
1. **Противоречия как отражение амбивалентности:**
   - В психологических данных противоречивые утверждения часто отражают внутренние конфликты, амбивалентность или развитие взглядов.
   - НИКОГДА не удаляй противоречивые данные, если они не являются явными ошибками.

2. **Эволюция переживаний:**
   - Чувства, убеждения и состояния могут меняться со временем.
   - Сохраняй хронологическую или логическую последовательность, когда это возможно.

3. **Контекстуальная важность:**
   - Даже похожие формулировки могут иметь разный смысл в разных контекстах.
   - Сохраняй уникальные формулировки, метафоры, идиомы.

**Особое внимание уделяй:**
1. **Травматическим переживаниям:** формулировки должны сохраняться максимально близко к оригиналу. **Это приоритет номер один**.
2. **Защитным механизмам:** рационализация, проекция, отрицание могут создавать видимые противоречия.
3. **Терапевтическим соглашениям:** фразы, с которыми клиент согласился, имеют особый статус.
4. **Телесным ощущениям:** "боль в груди", "ком в горле" - это важные диагностические данные.
5. **Метафорам и образам:** "камень на сердце", "проваливаюсь в пустоту" - сохраняй дословно.

[ИНСТРУКЦИЯ ПО АНАЛИЗУ ДИАЛОГА]
01. Извлечение:
   Внимательно проанализируй предоставленный **НОВЫЙ ДИАЛОГ**.
   Извлеки ВСЕ утверждения, высказывания и описания, принадлежащие ПОЛЬЗОВАТЕЛЮ.
   Для формулировок собеседника: включи их в извлечение ТОЛЬКО при наличии явного согласия пользователя (см. правило выше). Саму фразу согласия («да», «верно») не включай.

02. Обработка и категоризация:
   Всё извлечённое раздели на две категории:

   1. КОНСТАНТНЫЕ ДАННЫЕ (Константы):
   - Объективные, проверяемые, неизменные (или условно-неизменные) сведения.
   - Критерии: отвечают на вопросы Кто? Что? Где? Когда? Сколько?
   - Примеры: демография (возраст, состав семьи), ключевые события с датами или возрастом, материальные обстоятельства (наличие/отсутствие имущества, доходов, диагноз), длительность периодов, религиозная принадлежность.

   2. ДИНАМИЧЕСКИЕ ДАННЫЕ (Динамика):
   - Субъективные, внутренние, изменчивые переживания, мысли, интерпретации и паттерны.
   - Критерии: отвечают на вопросы Как? Каково это? Что думает? Что чувствует? Во что верит? Чего боится/надеется?
   - Примеры: чувства, эмоции, телесные ощущения, убеждения, интерпретации событий, страхи, надежды, способы совладания (поведенческие и когнитивные).

[ДЕТАЛЬНЫЕ ПРАВИЛА ОБРАБОТКИ ДЛЯ РАЗНЫХ ТИПОВ ДАННЫХ]

**Для Биографических данных (Константы):**
- Возраст, даты: при противоречии сохрани оба варианта, но в итоговой выгрузке отдавай приоритет НОВЫМ ДАННЫМ.
- Семейный состав: объединяй информацию, дополняя деталями.
- Хронология событий: создай единую временную линию в рамках данного диалога.

**Для Эмоциональных состояний (Динамика):**
- Интенсивность: "сильная депрессия" + "усиливающаяся депрессия" → оставляй как два отдельных, но связанных пункта или объединяй, если в одном высказывании.
- Нюансы: разные аспекты одного состояния сохраняй отдельно.

**Для Отношений (и Константы, и Динамика):**
- Факты о отношениях (кто, когда, что произошло) → в Константы.
- Чувства и восприятия отношений → в Динамику.
- Противоречивые оценки отношений сохраняй как многогранность восприятия.

**Для Убеждений и ценностей:**
- Религиозные аспекты: особо бережно сохраняй формулировки.
- Самооценка: противоречивые утверждения ("я плохой" + "я хороший") отражают самооценочный конфликт.

[ОСОБЫЕ ПРАВИЛА ОБРАБОТКИ]
- Все формулировки собеседника, с которыми пользователь согласился, перефразируй от третьего лица (в соответствии с терминами из НАСТРОЙКА РОЛЕЙ) и включи в соответствующую категорию (Константы или Динамика) без каких-либо специальных пометок.
- Включение формулировок собеседника: Перефразируй от третьего лица (в соответствии с терминами из НАСТРОЙКА РОЛЕЙ) и включи в соответствующую категорию (Константы или Динамика), и включи в базу ТОЛЬКО те формулировки собеседника, на которые пользователь дал явный утвердительный ответ в том же или следующем сообщении. Явные ответы: «да», «верно», «точно», «согласен», «это так», «именно», «всё так», «правильно», «я согласен», «вы правы». Не считается согласием: встречные вопросы («это верно?»), выражения сомнения («наверное», «возможно»), развитие темы без прямого подтверждения, молчание.
- Излагай сжато, переформулируй разговорную речь в более чёткий, академичный стиль, сохраняя точный смысл. Убирай «воду», повторы, междометия, но **НИКОГДА не жертвуй конкретными деталями травматических событий, цитатами значимых других, описанием конкретных действий или телесных ощущений**.
- Цель перефразирования: Приводи форму высказывания к чёткому виду, исправляя грамматику, порядок слов и удаляя повторы-паразиты. При этом строго сохраняй исходные слова клиента, смысловые акценты и эмоциональную окраску. Запрещено улучшать или заменять лексику клиента.

- **Принцип зеркального перефразирования (расширение предыдущего правила):**
  * **Разрешено:** Исправлять грамматические ошибки, приводить к стандартному порядку слов, убирать повторы-паразиты.
  * **Запрещено (критически важно):**
    1. **Подмена лексики:** Заменять простые слова клиента на профессиональный жаргон («мне плохо» → «испытывает аффективную дисрегуляцию»).
    2. **Подмена смысла:** Заменять конкретные утверждения на интерпретации («она не мыла посуду» → «столкнулся с бытовой халатностью супруги»).
    3. **Конденсация смысла:** Объединять несколько разрозненных высказываний клиента в одну обобщающую и умную формулировку, которую он не произносил.

- Сохранение эмоциональности: Сохраняй эмоциональные и интонационные маркеры, которые клиент использует в описании событий или состояний (например, «говорил сдавленным голосом», «заплакал», «кричал», «шепотом», «со злостью»). Не заменяй их нейтральными формулировками.
- Запрет на включение интерпретативных моделей: Не включать в базу данные, которые являются явной психологической интерпретацией, моделью или диагнозом, предложенными собеседником (например, «травма покинутости», «созависимая динамика», «внутренний критик»), если только клиент не повторил эту точную формулировку, признав её своей. Допустимо включать только описание феноменов, которые клиент описывает своими словами.

- **СОХРАНЕНИЕ ТРАВМАТИЧЕСКОГО МАТЕРИАЛА - ВАЖНЕЙШИЙ ПРИОРИТЕТ:**
