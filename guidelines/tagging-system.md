# Система тегов

Теги используются в мета-блоке каждого промпта или сниппета для классификации,
поиска и фильтрации. Все теги начинаются с символа `#` и записываются в нижнем регистре.


============================================================
## Структура тега

Каждый тег принадлежит к одной из четырёх категорий:


------------------------------
### 1. Тип содержимого
Указывает, чем является файл по своей сути.

- `#prompt` — полноценный, готовый к использованию промпт.
- `#snippet` — фрагмент, заготовка, «строительный блок» для промптов.
- `#meta` — правила, инструкции, заметки для человека (обычно в `.guide`-файлах).


------------------------------
### 2. Функциональная область
Описывает назначение или область применения.

- `#role` — ролевая инструкция (эксперт, ассистент, персонаж).
- `#system` — системный промпт, определяющий поведение LLM.
- `#psychology` — психологические, терапевтические сценарии.
- `#text` — работа с текстом: генерация, обработка, анализ.
- `#code` — программирование, разработка, code review.
- `#research` — поиск информации, фактчекинг.
- `#formatting` — управление форматом вывода LLM.
- `#context` — управление контекстом (исключение, приоритизация).
- `#jailbreak` — техники обхода ограничений (если применимо).


------------------------------
### 3. Статус
Показывает готовность промпта к использованию.

- `#draft` — черновик, требуется доработка.
- `#untested` — не проверен на практике.
- `#tested` — проверен на нескольких моделях, результаты удовлетворительные.
- `#ready` — полностью готов и рекомендован к использованию.
- `#archived` — устарел, сохранён для истории.

Статусы можно комбинировать, например `#untested, #draft`.


==============================
### 4. Дополнительные (опционально)
Любые уточняющие теги, которые помогают при поиске.

- `#llm` — относится непосредственно к LLM (промпт для ИИ).
- `#persona` — создание определённой личности/персонажа.
- `#memory` — работа с памятью диалога.
- и другие по мере необходимости.


============================================================
## Правила использования

- Теги перечисляются в строке `tags:` через запятую.
- Порядок тегов не регламентирован, но рекомендуется: тип содержимого → область → статус.
- Допускается любое количество тегов, но минимально — один тип содержимого.
- Новые теги добавляются в этот файл перед первым использованием, чтобы сохранить единообразие.


============================================================
## Примеры

```yaml
tags: [#prompt, #role, #psychology, #tested]
tags: [#snippet, #formatting, #untested, #draft]
tags: [#prompt, #code, #jailbreak, #draft]
tags: [#prompt, #research, #ready]
tags: [#snippet, #context]
```


============================================================
## Расширение системы
При появлении новой предметной области, для которой нет подходящего тега,
следует добавить его в раздел «Функциональная область» этого файла,
сохраняя стиль: нижний регистр, осмысленное английское слово или короткое словосочетание.
